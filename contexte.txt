Titre du projet : Veille et benchmark sur les services d’analyse de sentiments proposés par Hugging Face

Contexte :
Dans le cadre de ma certification, je dois réaliser un cas pratique de veille technologique (Bloc de compétence 2 - E2). L'objectif est d'identifier un besoin d’intelligence artificielle et de proposer un service IA existant répondant à ce besoin, en réalisant une veille structurée et un benchmark. J’ai choisi de travailler sur l’analyse automatique de sentiments dans des textes (avis clients, commentaires, publications réseaux sociaux…).

Objectif fonctionnel :
Proposer un service d’intelligence artificielle capable de détecter le sentiment d’un texte court (positif, neutre, négatif) afin d’automatiser l’analyse d’avis utilisateurs.

Objectif technique :
Identifier, comparer et configurer plusieurs modèles d’analyse de sentiments disponibles sur la plateforme Hugging Face. Tester leur usage via l’API, et documenter les différences en termes de performance, accessibilité et pertinence des résultats.

Compétences visées :
- C6 : Organisation d'une veille technologique sur les modèles de NLP.
- C7 : Benchmark de plusieurs services IA existants sur Hugging Face.
- C8 : Paramétrage simple d’un modèle IA via l’API de Hugging Face.

Plan d’action demandé à l’IA :
1. Sélectionner 3 modèles Hugging Face adaptés à l’analyse de sentiments sur des textes en français et anglais.
   ➤ Exemples : 
     - `distilbert-base-uncased-finetuned-sst-2-english`
     - `nlptown/bert-base-multilingual-uncased-sentiment`
     - `cardiffnlp/twitter-roberta-base-sentiment`

2. Pour chaque modèle, collecter les informations suivantes :
   - Langues supportées
   - Précision attendue (si benchmark public ou papier)
   - Poids du modèle (en Mo)
   - Temps moyen de réponse
   - Simplicité d'intégration via API (exemples en Python)
   - Coût (si usage via Inference API Hugging Face)
   - Accessibilité et documentation

3. Réaliser un test simple sur un jeu d'exemples (5 phrases en anglais, 5 en français) et comparer les résultats entre les modèles.

4. Fournir un tableau comparatif clair des 3 modèles selon les critères ci-dessus.

5. Générer un fichier Python prêt à l’emploi permettant de tester les 3 modèles en local (via pipeline Transformers) ou via l’API Hugging Face.

6. Générer une courte synthèse de veille (1 page) au format texte brut ou Markdown, conforme aux standards d’accessibilité (valeurs contrastées, structuration lisible).

Contraintes :
- Ne pas utiliser de services payants ou nécessitant une authentification complexe.
- L’API de Hugging Face peut être utilisée gratuitement jusqu’à 30 requêtes/jour sans token.
- Le code final doit être simple, bien commenté, et exécutable depuis un script.

Livrables attendus :
- Script Python de test (3 modèles comparés).
- Fichier texte ou Markdown avec le tableau comparatif.
- Synthèse de veille sur les outils de sentiment analysis disponibles.
- Capture d’écran ou démonstration éventuelle des résultats.

But final :
Prouver la capacité à organiser une veille ciblée, comparer plusieurs services IA existants, et en configurer un pour un cas d’usage réel.
